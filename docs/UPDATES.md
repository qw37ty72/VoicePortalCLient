# Автообновление Voice Portal

## Как это работает

1. **При каждом запуске** приложение проверяет на GitHub наличие новой версии (через electron-updater).
2. Если обновление есть — оно **скачивается в фоне**. По окончании показывается окно: «Перезапустите приложение».
3. После перезапуска устанавливается новая версия. Другу ничего нажимать не нужно, кроме «Перезапустить сейчас» (или он может закрыть приложение — тогда обновление встанет при следующем запуске).

## Что сделать один раз (тебе)

1. **Создай репозиторий на GitHub** (если ещё нет) и запушь туда проект.

2. **Укажи свой репозиторий в `client/package.json`:**
   - В блоке `"build"."publish"` замени `YOUR_GITHUB_USERNAME` на свой логин GitHub, а `VoicePortal` — на имя репозитория (если другое).
   - В блоке `"repository"` замени URL на свой, например:  
     `"url": "https://github.com/ТВОЙ_ЛОГИН/VoicePortal.git"`

3. **Первый релиз вручную (один раз):**
   - Собери приложение: из корня проекта `npm run build -w client`
   - В папке `client/release/` появятся установщики. Один из них (например, `.exe` для Windows) отдай другу для установки.

Дальше обновления будут через систему ниже.

## Как выкатывать новую версию (обновление для друга)

1. Вносишь изменения в код, коммитишь в Git.
2. Создаёшь и пушишь **тег** с версией, например:
   ```bash
   git tag v1.0.1
   git push origin v1.0.1
   ```
3. На GitHub в разделе **Actions** запустится workflow «Build and release»: соберётся приложение и создастся новый **Release** с установщиками.
4. У друга при **следующем запуске** приложения появится проверка обновлений; если вышла новая версия — скачается и предложится перезапуск.

Версию в теге лучше совпадающую с версией в `client/package.json` (например, перед `git tag v1.0.1` можно сделать в `client` версию 1.0.1). Workflow при сборке по тегу сам подставляет версию из тега в сборку.
